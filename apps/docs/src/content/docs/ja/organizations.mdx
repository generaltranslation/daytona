DaytonaのOrganizationsは、リソースをグループ化し、他のユーザーと簡単に協力するための方法です。
Organizationのユーザーは、権限に応じて、Sandboxes、Snapshots、Registriesなどの同じ主要リソースセットを表示・管理し、これらのリソースに対して同じクォータを消費します。

Daytonaにサインアップした後、ユーザーにはPersonal Organizationが割り当てられます。このデフォルトのOrganizationは新しいメンバーを受け入れることができず、削除することもできません。独立した請求システムを持ち、最も一般的にはテスト環境として、またはソロ開発者によって使用されます。組織間の切り替えは、サイドバー上部のドロップダウンメニューからオプションを選択することで行います。

## 組織の役割

組織内のユーザーは、`Owner`と`Member`という2つの異なる**役割**のいずれかを持つことができます。`Owner`は組織とそのリソースに対する完全な管理アクセス権を持ちます。`Member`は組織に対する管理アクセス権を持たず、組織リソースへのアクセスは**割り当て**に基づきます。

### 管理アクション

組織の`Owner`は以下のような管理アクションを実行できます：

- 組織に新しいユーザーを招待する
- 保留中の招待を管理する
- 組織内のユーザーの役割を変更する
- 組織メンバーの割り当てを更新する
- 組織からユーザーを削除する
- 組織を削除する

## 新しいユーザーの招待

Organization `Owner`として、あなたのOrganizationに新しいユーザーを招待するには、_Membersページ_に移動し、
_Invite Member_をクリックして、招待したいユーザーのメールアドレスを入力し、**Role**を選択します。
`Member`ロールを選択した場合、その**Assignments**も定義できます。

## 利用可能な割り当て

利用可能な**割り当て**のリストには以下が含まれます：

| 割り当て                    | 説明                                                         |
| ----------------------- | ------------------------------------------------------------------- |
| **`Viewer (required)`** | 組織内のすべてのリソースへの読み取りアクセスを付与             |
| **`Developer`**         | 組織内でサンドボックスとキーを作成する機能を付与 |
| **`Sandboxes Admin`**   | 組織内のサンドボックスへの管理者アクセスを付与                |
| **`Snapshots Admin`**   | 組織内のスナップショットへの管理者アクセスを付与                |
| **`Registries Admin`**  | 組織内のレジストリへの管理者アクセスを付与               |
| **`Super Admin`**       | 組織内のすべてのリソースへの完全なアクセスを付与             |

## 招待の管理

他の組織への参加の保留中の招待を表示するには、ユーザーはサイドバーの下部にあるドロップダウンを展開し、_招待_をクリックして_招待ページ_に移動できます。ユーザーが組織への参加招待を受け入れると、その組織に割り当てられたリソースクォータにアクセスできるようになり、新しいAPIキーを発行してサンドボックスを作成することで続行できます。

## 設定

ダッシュボードの設定サブページでは、組織IDと名前を確認し、不要になった場合は組織を削除することができます。この操作は元に戻すことができませんので、慎重に進めてください。個人組織はデフォルトで存在し、削除することはできません。
